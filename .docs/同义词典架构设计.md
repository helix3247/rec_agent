# 同义词典架构设计：从 MVP 到超大规模

> 指导原则：同义词典不是一成不变的配置文件，而是**动态的、可管理的、数据驱动的系统**。
> 本文档展示三个渐进式方案，从本地文件到分布式中心，以及对应的面试话术。

---

## 核心问题

**为什么要加同义词典？**

电商中同一个商品，用户用不同方式搜索：
```
卫衣 = 帽衫 = 连帽衫 = 套头衫
运动鞋 = 跑鞋 = 跑步鞋
高跟鞋 = 细跟鞋 = 高根鞋
```

从我们的 NDCG 评测数据可以看出威力：

| Query | BM25 (无同义词) | Hybrid+RRF (有同义词) |
|-------|-------------|-------------|
| "秋天穿的连帽衫" | 0.0 ❌ | 1.0 ✓ |
| "拍照好的手机" | 0.0 ❌ | 1.0 ✓ |
| "华为手机" | 0.0 ❌ | 1.0 ✓ |

**同义词的三大价值**：
1. **提升 Recall**：用户搜"帽衫"能找到标签为"卫衣"的商品
2. **降低成本**：BM25 + 同义词（16ms）比 KNN 单独（603ms）快 40 倍
3. **面试加分**：展现从"被动用向量搜索"到"主动设计权衡方案"的工程思维

## 方案 1：本地文件 + 定期更新（MVP 方案）

### 适用场景
- 日均检索量 < 100 万
- 同义词变化不频繁（周级更新）
- 团队小，运维成本要求低

### 技术实现

**当前项目的实现**：
```python
# app/tools/search.py
```

**更新流程**：
```bash
# 编辑 synonym.txt

# 提交代码
git add ./volumes/elasticsearch-config/synonym.txt
git commit -m "feat: add new synonyms for home appliances"

# 部署（使用 updateable=true，无需重启）
docker-compose restart elasticsearch

# 或在线刷新（ES 命令）
curl -X POST "localhost:9200/product_index/_refresh"
```

### 优缺点

**优点**：
- ✅ 简单，无额外依赖和复杂度
- ✅ 性能最好（同义词预加载到内存）
- ✅ 成本最低
- ✅ 便于版本管理（Git 自动追踪）

**缺点**：
- ❌ 更新延迟（手动编辑 + 部署）
- ❌ 无实时性（无法应对突发需求，如"双 11 新品类")
- ❌ 无审计日志（谁改的、什么时候改的不清楚）
- ❌ 运营不能自助（需要技术人员介入）

### 面试话术

> "对于 MVP 阶段，我采用本地 synonym.txt + 定期刷新的方案。通过 ES 9.x 的 `updateable: true` 特性，可以在无需重启索引的情况下更新同义词。这保证了快速迭代，但缺点是更新不够实时。在生产环境规模化之后，我会演进到方案 2。"

## 方案 2：同义词中心（推荐，大规模生产）

### 适用场景
- 日均检索量 100 万 ~ 亿级
- 同义词需要频繁更新（天级、小时级）
- 有运营团队维护

### 架构设计

```
┌─────────────────┐
│  同义词管理中心  │
│  (内部系统)      │
└────────┬────────┘
         │ 定时同步/推送
         ↓
┌─────────────────┐     ┌──────────────┐
│  Redis/Memcache │────→│  Elasticsearch│
│  (热缓存)        │     │  (Synonym词库)│
└─────────────────┘     └──────────────┘
         ↑
    ┌────┴────┬──────────┬──────────┐
    │          │          │          │
   运营      产品      数据分析    爬虫学习
  (上传)    (标签)     (推荐)     (日志分析)
```

### 数据模型

**同义词表设计**：
```sql
CREATE TABLE synonym_dict (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    canonical_term VARCHAR(255),      -- "卫衣" (标准词)
    synonym_term VARCHAR(255),        -- "帽衫", "连帽衫"
    category VARCHAR(50),             -- "服装"
    confidence FLOAT DEFAULT 1.0,     -- 0.95 (可信度，用于排序)
    source ENUM(                      -- 来源追踪
        'manual',                     -- 运营手动添加
        'data_driven',                -- 数据驱动自动推荐
        'user_feedback'               -- 用户反馈
    ),
    priority INT DEFAULT 0,           -- 优先级（高频词靠前）
    created_by VARCHAR(100),          -- 谁添加的
    updated_by VARCHAR(100),
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    is_active BOOLEAN DEFAULT TRUE,
    version INT DEFAULT 1,
    COMMENT '同义词库，支持版本管理和审计'
);

-- 索引优化
CREATE INDEX idx_canonical ON synonym_dict(canonical_term, is_active);
CREATE INDEX idx_term ON synonym_dict(synonym_term, is_active);
CREATE INDEX idx_source ON synonym_dict(source, created_at);
CREATE INDEX idx_category ON synonym_dict(category);
```

**变更历史表（审计）**：
```sql
CREATE TABLE synonym_dict_history (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    dict_id BIGINT NOT NULL,
    action ENUM('INSERT', 'UPDATE', 'DELETE'),
    old_value JSON,
    new_value JSON,
    changed_by VARCHAR(100),
    changed_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    reason VARCHAR(500),
    COMMENT '审计日志，支持回滚'
);
```

### 后端服务实现

```python
# app/services/synonym_service.py

from typing import Optional, List
from datetime import datetime, timedelta
import json
from app.core.db_pool import DBPool
from app.core.cache import redis_client
from app.core.logger import get_logger

_logger = get_logger(agent_name="SynonymService")


class SynonymService:
    """同义词管理服务"""
    
    CACHE_TTL = 86400  # 24 小时
    CACHE_PREFIX = "synonym:"
    
    def __init__(self):
        self.db = DBPool.get_pool()
        self.redis = redis_client
    
    # ─── 查询接口 ───
    
    async def get_synonyms(self, term: str) -> List[str]:
        """
        查询某个词的所有同义词。
        优先查缓存，缓存失效后查 DB。
        """
        cache_key = f"{self.CACHE_PREFIX}{term}"
        
        # 先查 Redis
        cached = self.redis.get(cache_key)
        if cached:
            _logger.debug(f"Synonym cache hit: {term}")
            return json.loads(cached)
        
        # 查数据库
        async with self.db.acquire() as conn:
            async with conn.cursor() as cur:
                await cur.execute(
                    "SELECT synonym_term FROM synonym_dict "
                    "WHERE canonical_term = %s AND is_active = TRUE "
                    "ORDER BY confidence DESC",
                    [term]
                )
                rows = await cur.fetchall()
        
        synonyms = [row[0] for row in rows]
        
        # 存入缓存
        if synonyms:
            self.redis.setex(
                cache_key,
                self.CACHE_TTL,
                json.dumps(synonyms)
            )
        
        _logger.info(f"Loaded synonyms: {term} → {synonyms}")
        return synonyms
    
    async def get_reverse_synonyms(self, term: str) -> List[str]:
        """
        查询反向同义词：谁把这个词当作同义词。
        例如：搜"帽衫"，返回 ["卫衣"] (卫衣的同义词包括帽衫)
        """
        async with self.db.acquire() as conn:
            async with conn.cursor() as cur:
                await cur.execute(
                    "SELECT DISTINCT canonical_term FROM synonym_dict "
                    "WHERE synonym_term = %s AND is_active = TRUE",
                    [term]
                )
                rows = await cur.fetchall()
        
        return [row[0] for row in rows]
    
    # ─── 管理接口（运营后台使用） ───
    
    async def add_synonym(
        self,
        canonical: str,
        synonym: str,
        category: str,
        source: str = "manual",
        confidence: float = 1.0,
        created_by: str = "admin",
        reason: str = ""
    ) -> dict:
        """
        新增同义词。
        运营可以通过后台调用此接口。
        """
        try:
            async with self.db.acquire() as conn:
                async with conn.cursor() as cur:
                    # 插入同义词
                    await cur.execute(
                        """
                        INSERT INTO synonym_dict 
                        (canonical_term, synonym_term, category, source, 
                         confidence, created_by)
                        VALUES (%s, %s, %s, %s, %s, %s)
                        """,
                        [canonical, synonym, category, source, confidence, created_by]
                    )
                    
                    # 插入审计日志
                    await cur.execute(
                        """
                        INSERT INTO synonym_dict_history
                        (dict_id, action, new_value, changed_by, reason)
                        VALUES (LAST_INSERT_ID(), 'INSERT', %s, %s, %s)
                        """,
                        [
                            json.dumps({
                                "canonical": canonical,
                                "synonym": synonym,
                                "confidence": confidence
                            }),
                            created_by,
                            reason
                        ]
                    )
                    
                    await conn.commit()
            
            # 清除缓存
            self.redis.delete(f"{self.CACHE_PREFIX}{canonical}")
            
            # 异步同步到 ES
            await self._sync_to_elasticsearch()
            
            _logger.info(
                f"Add synonym: {canonical} → {synonym} "
                f"(source={source}, by={created_by})"
            )
            
            return {
                "status": "ok",
                "message": f"已添加同义词: {canonical} → {synonym}"
            }
        
        except Exception as e:
            _logger.error(f"Failed to add synonym: {e}")
            return {"status": "error", "message": str(e)}
    
    async def disable_synonym(
        self,
        dict_id: int,
        updated_by: str = "admin",
        reason: str = ""
    ) -> dict:
        """禁用同义词（逻辑删除）"""
        try:
            async with self.db.acquire() as conn:
                async with conn.cursor() as cur:
                    # 获取旧值
                    await cur.execute(
                        "SELECT canonical_term, synonym_term FROM synonym_dict "
                        "WHERE id = %s",
                        [dict_id]
                    )
                    row = await cur.fetchone()
                    
                    if not row:
                        return {"status": "error", "message": "ID not found"}
                    
                    canonical, synonym = row
                    
                    # 更新状态
                    await cur.execute(
                        "UPDATE synonym_dict SET is_active = FALSE "
                        "WHERE id = %s",
                        [dict_id]
                    )
                    
                    # 插入审计日志
                    await cur.execute(
                        """
                        INSERT INTO synonym_dict_history
                        (dict_id, action, new_value, changed_by, reason)
                        VALUES (%s, 'UPDATE', %s, %s, %s)
                        """,
                        [
                            dict_id,
                            json.dumps({"is_active": False}),
                            updated_by,
                            reason
                        ]
                    )
                    
                    await conn.commit()
            
            # 清除缓存
            self.redis.delete(f"{self.CACHE_PREFIX}{canonical}")
            
            # 同步到 ES
            await self._sync_to_elasticsearch()
            
            return {"status": "ok", "message": "已禁用同义词"}
        
        except Exception as e:
            _logger.error(f"Failed to disable synonym: {e}")
            return {"status": "error", "message": str(e)}
    
    # ─── 同步接口 ───
    
    async def _sync_to_elasticsearch(self) -> None:
        """
        将数据库中的同义词导出为 synonym.txt 并推送到 ES。
        可以是定时任务或事件驱动触发。
        """
        try:
            # Step 1: 从 DB 读取所有活跃同义词
            async with self.db.acquire() as conn:
                async with conn.cursor() as cur:
                    await cur.execute(
                        """
                        SELECT canonical_term, synonym_term 
                        FROM synonym_dict
                        WHERE is_active = TRUE
                        ORDER BY canonical_term, confidence DESC
                        """
                    )
                    rows = await cur.fetchall()
            
            # Step 2: 生成 synonym.txt 格式
            # 格式："canonical_term,synonym1,synonym2,synonym3"
            synonym_map = {}
            for canonical, synonym in rows:
                if canonical not in synonym_map:
                    synonym_map[canonical] = []
                synonym_map[canonical].append(synonym)
            
            synonym_content = "\n".join(
                f"{canonical},{','.join(synonyms)}"
                for canonical, synonyms in sorted(synonym_map.items())
            )
            
            # Step 3: 上传到 S3 或 ConfigMap
            import boto3
            s3 = boto3.client('s3')
            s3.put_object(
                Bucket='rec-agent-config',
                Key='elasticsearch/synonym.txt',
                Body=synonym_content.encode('utf-8'),
                ContentType='text/plain'
            )
            
            _logger.info(
                f"Synced {len(synonym_map)} synonyms to S3 "
                f"({sum(len(v) for v in synonym_map.values())} relations)"
            )
            
            # Step 4: 通知 ES 刷新（或在下次 refresh 时生效）
            # 注：updateable=true 会自动加载新文件
            
        except Exception as e:
            _logger.error(f"Failed to sync synonyms to ES: {e}")
    
    async def batch_import_synonyms(self, file_path: str) -> dict:
        """
        批量导入同义词（Excel/CSV）。
        格式：canonical_term, synonym_term, category, confidence
        """
        import pandas as pd
        
        try:
            df = pd.read_csv(file_path)
            
            async with self.db.acquire() as conn:
                async with conn.cursor() as cur:
                    for _, row in df.iterrows():
                        await cur.execute(
                            """
                            INSERT INTO synonym_dict
                            (canonical_term, synonym_term, category, 
                             confidence, source, created_by)
                            VALUES (%s, %s, %s, %s, 'manual', 'batch_import')
                            """,
                            [
                                row['canonical_term'],
                                row['synonym_term'],
                                row.get('category', 'general'),
                                row.get('confidence', 1.0)
                            ]
                        )
                    
                    await conn.commit()
            
            # 清除所有缓存
            for key in self.redis.scan_iter(match=f"{self.CACHE_PREFIX}*"):
                self.redis.delete(key)
            
            # 同步到 ES
            await self._sync_to_elasticsearch()
            
            _logger.info(f"Batch imported {len(df)} synonyms")
            return {"status": "ok", "count": len(df)}
        
        except Exception as e:
            _logger.error(f"Failed to batch import: {e}")
            return {"status": "error", "message": str(e)}


class AutoSuggestService:
    """
    数据驱动的同义词推荐服务。
    从搜索日志分析出"搜 A 无结果、搜 B 有结果"的模式，
    自动推荐为同义词对，由运营人工审核。
    """
    
    def __init__(self):
        self.db = DBPool.get_pool()
        self.synonym_service = SynonymService()
    
    async def suggest_from_search_logs(self, days: int = 7) -> List[dict]:
        """
        分析搜索日志，自动推荐新同义词。
        """
        try:
            async with self.db.acquire() as conn:
                async with conn.cursor() as cur:
                    # 查询零结果的搜索
                    await cur.execute(
                        """
                        SELECT search_term, COUNT(*) as freq
                        FROM search_log
                        WHERE result_count = 0
                          AND created_at >= DATE_SUB(NOW(), INTERVAL %s DAY)
                        GROUP BY search_term
                        HAVING freq > 1
                        ORDER BY freq DESC
                        LIMIT 100
                        """,
                        [days]
                    )
                    
                    zero_result_queries = await cur.fetchall()
            
            suggestions = []
            
            for query, freq in zero_result_queries:
                # 用向量搜索找"最相似的已检索成功的词"
                # 这里简化为查找相似的已存在的同义词对
                async with self.db.acquire() as conn:
                    async with conn.cursor() as cur:
                        await cur.execute(
                            """
                            SELECT canonical_term, COUNT(*) as hits
                            FROM search_log sl
                            LEFT JOIN synonym_dict sd 
                              ON (sl.search_term = sd.canonical_term 
                                  OR sl.search_term = sd.synonym_term)
                            WHERE sl.result_count > 0
                              AND sd.canonical_term IS NOT NULL
                              AND LEVENSHTEIN(sl.search_term, %s) < 3
                            GROUP BY canonical_term
                            ORDER BY hits DESC
                            LIMIT 1
                            """,
                            [query]
                        )
                        
                        candidate = await cur.fetchone()
                
                if candidate:
                    canonical, hits = candidate
                    suggestions.append({
                        "canonical": canonical,
                        "synonym": query,
                        "frequency": freq,
                        "confidence": min(0.95, 0.5 + freq * 0.1),
                        "reason": f"零结果 {freq} 次，相似度高"
                    })
            
            _logger.info(f"Auto-suggested {len(suggestions)} new synonyms")
            return suggestions
        
        except Exception as e:
            _logger.error(f"Failed to suggest synonyms: {e}")
            return []
```

### 优缺点

**优点**：

### 面试话术

> "在大规模生产环境，我会建立一个同义词中心系统。
>

> 
> 这样既保证了灵活性和实时性，又有完整的审计链路，适合中到大规模的电商平台。"

## 方案 3：混合策略（终极方案）

### 适用场景
- 日均检索量 > 亿级
- 需要毫秒级的实时性
- 对成本和性能都有严格要求

### 分层架构

```
实时层 (毫秒级)        ← Redis 缓存热词同义词
    ↓
召回层 (秒级)          ← BM25 + IK + 本地同义词 + 扩展词
    ↓
精排层 (百毫秒级)      ← 向量模型重排（只重排 Top 100）
    ↓
业务决策 (秒级)        ← LLM Agent 学习用户反馈
```

### 技术实现

```python
# app/services/search_service.py

from enum import Enum
from typing import List, Optional
import asyncio

class SynonymMode(Enum):
    BATCH = "batch"          # 依赖 ES 内部 synonym.txt
    REALTIME = "realtime"    # Redis 动态查询
    HYBRID = "hybrid"        # 先 Redis 再 ES（推荐）


class HybridSearchService:
    """
    混合搜索服务：兼顾实时性和性能。
    """
    
    def __init__(self, mode: SynonymMode = SynonymMode.HYBRID):
        self.mode = mode
        self.synonym_service = SynonymService()
        self.es_service = ESService()
        self.embedding_service = EmbeddingService()
    
    async def search(
        self,
        query: str,
        top_k: int = 10,
        strategy: str = "hybrid_rrf",
        enable_semantic_rerank: bool = True,
    ) -> List[dict]:
        """
        混合搜索入口。
        
        Args:
            query: 用户查询
            top_k: 返回数量
            strategy: 检索策略 ("bm25" | "knn" | "hybrid_rrf")
            enable_semantic_rerank: 是否对 Top 100 做向量重排
        """
        
        # Step 1: 查询词扩展（同义词）
        if self.mode in (SynonymMode.REALTIME, SynonymMode.HYBRID):
            expanded_terms = await self._expand_query_realtime(query)
        else:
            expanded_terms = [query]  # ES 内部处理
        
        # Step 2: 并行执行多路检索
        if strategy == "bm25":
            results = await self._search_bm25(expanded_terms, top_k)
        
        elif strategy == "knn":
            results = await self._search_knn(query, top_k)
        
        elif strategy == "hybrid_rrf":
            # 并行执行两路检索
            bm25_task = self._search_bm25(expanded_terms, top_k)
            knn_task = self._search_knn(query, top_k)
            
            bm25_results, knn_results = await asyncio.gather(
                bm25_task, knn_task, return_exceptions=True
            )
            
            # 错误处理
            if isinstance(bm25_results, Exception):
                bm25_results = []
            if isinstance(knn_results, Exception):
                knn_results = []
            
            # RRF 融合
            results = self._rrf_merge(bm25_results, knn_results, top_k)
        
        # Step 3: 可选的向量重排（精排）
        # 场景：结果质量要求高，但对延迟容忍度较低
        if enable_semantic_rerank and len(results) > 10:
            results = await self._semantic_rerank(
                query, results[:100], keep_top_k=top_k
            )
        
        return results
    
    async def _expand_query_realtime(self, query: str) -> List[str]:
        """实时从 Redis 扩展同义词"""
        terms = [query]
        
        # 先查直接同义词
        synonyms = await self.synonym_service.get_synonyms(query)
        terms.extend(synonyms)
        
        # 再查反向同义词
        reverse = await self.synonym_service.get_reverse_synonyms(query)
        terms.extend(reverse)
        
        # 限制扩展数量，避免笛卡尔爆炸
        # 只保留 Top-3 同义词（按热度）
        return list(dict.fromkeys(terms))[:4]  # 原词 + 3 个同义词
    
    async def _search_bm25(
        self, expanded_terms: List[str], top_k: int
    ) -> List[dict]:
        """执行 BM25 搜索"""
        # 对多个词并行检索，然后合并
        tasks = [
            self.es_service.search_bm25(term, top_k * 3)
            for term in expanded_terms
        ]
        
        all_results = await asyncio.gather(*tasks)
        
        # 合并并去重（按 score 和出现频率）
        merged = self._merge_and_deduplicate(all_results)
        return merged[:top_k]
    
    async def _search_knn(self, query: str, top_k: int) -> List[dict]:
        """执行 KNN 向量检索"""
        query_vector = await self.embedding_service.embed(query)
        return await self.es_service.search_knn(query_vector, top_k)
    
    def _rrf_merge(
        self,
        bm25_results: List[dict],
        knn_results: List[dict],
        top_k: int,
        k: int = 60,
        bm25_weight: float = 0.4,
        knn_weight: float = 0.6,
    ) -> List[dict]:
        """显式 RRF 融合"""
        scores = {}
        docs = {}
        
        for rank, hit in enumerate(bm25_results, start=1):
            pid = hit['product_id']
            scores[pid] = scores.get(pid, 0.0) + bm25_weight / (k + rank)
            docs[pid] = hit
        
        for rank, hit in enumerate(knn_results, start=1):
            pid = hit['product_id']
            scores[pid] = scores.get(pid, 0.0) + knn_weight / (k + rank)
            if pid not in docs:
                docs[pid] = hit
        
        sorted_pids = sorted(scores, key=scores.__getitem__, reverse=True)
        return [
            {**docs[pid], "rrf_score": round(scores[pid], 6)}
            for pid in sorted_pids[:top_k]
        ]
    
    async def _semantic_rerank(
        self, query: str, candidates: List[dict], keep_top_k: int
    ) -> List[dict]:
        """
        语义重排：对候选结果用向量模型重新打分。
        成本较高，只在对质量要求特别高的场景使用。
        """
        query_vector = await self.embedding_service.embed(query)
        
        reranked = []
        for doc in candidates:
            # 计算与 query 的相似度
            doc_text = f"{doc['name']} {doc['description']}"
            doc_vector = await self.embedding_service.embed(doc_text)
            
            similarity = self.embedding_service.cosine_similarity(
                query_vector, doc_vector
            )
            
            doc['semantic_score'] = similarity
            reranked.append(doc)
        
        # 按语义相似度排序
        reranked.sort(key=lambda x: x['semantic_score'], reverse=True)
        return reranked[:keep_top_k]
```

### 成本对比

```
每日 10 亿次搜索（日均峰值 100 万 QPS）的成本估算：

1. BM25 纯本地：
   - 服务器成本：60 万
   - 同义词人工维护：5 人月 = 4 万

2. 方案 2（同义词中心）：
   - 额外 MySQL：10 万
   - 额外 Redis 集群：20 万
   - 工程维护：2 人月 = 1.5 万
   - 总成本：~92 万（+50%）

3. 方案 3（混合）：
   - 额外基础设施：30 万
   - 向量模型推理成本：100 万（用第三方 API）
   - 总成本：190 万（+200%）

结论：方案 3 多花 2 倍钱，但精准率从 83% 提升到 94%，
      投资回报率（提升转化率）通常能覆盖额外成本。
```

## 总结

同义词典的核心原则：
1. **从简单开始**：本地文件足以应付 MVP
2. **逐步演进**：不要一开始就过度设计
3. **可观测可管理**：有审计日志、版本控制
4. **数据驱动**：用数据而不是主观判断来扩展同义词
5. **性能与灵活性的权衡**：实时性越强，成本越高

这个思维方式本身就是大厂期望的"**工程文化**"。
