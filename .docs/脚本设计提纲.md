# 数据生成与入库脚本提纲

根据 [技术方案.md](file:///d:/Project/rec_agent/.docs/技术方案.md) 的混合架构 (MySQL + ES + Milvus)，我们需要在 `scripts/` 目录下创建以下 3 个核心脚本：

## 1. `scripts/generate_mock_data.py` (数据生成器)
**目标**: 生成一份包含用户、商品、评论的“黄金源数据” (JSON格式)，作为所有数据库的输入。

*   **造数逻辑**:
    *   **商品数据 (Products)**:
        *   品类: 覆盖 "数码相机" (高客单价、参数复杂) 和 "男装" (非标品、注重搭配)。
        *   数量: 生成 50-100 个 SKU。
        *   字段: `id`, `name`, `category`, `price`, `brand`, `specs` (JSON: 像素/材质), `tags` (复古/商务), `description` (用于Embedding)。
        *   *逻辑*: 使用预定义的品牌列表和形容词模板随机组合生成，如 "Sony A7M4 全画幅微单" 或 "Uniqlo 商务休闲衬衫"。
    *   **用户数据 (Users)**:
        *   数量: 10 个典型用户画像。
        *   字段: `user_id`, `name`, `age`, `gender`, `budget_level` (high/mid/low), `style_preference`。
    *   **非结构化数据 (Reviews/Docs)**:
        *   数量: 每个商品生成 3-5 条评论或说明书片段。
        *   内容: 针对 RAG 场景，生成带有细节的文本，如 "这款相机的夜景噪点控制得很好，但在强光下容易过曝"。

## 2. `scripts/init_mysql.py` (元数据入库)
**目标**: 初始化 MySQL 表结构，并将 Mock 数据写入。

*   **逻辑**:
    1.  连接 MySQL (Docker)。
    2.  创建表: `users`, `products`, `interactions` (空表), `orders` (空表)。
    3.  读取 `mock_data.json`。
    4.  执行 `INSERT INTO` 语句批量写入用户和商品数据。

## 3. `scripts/sync_to_vector_db.py` (向量化与索引同步)
**目标**: 将商品数据同步到 Elasticsearch (用于搜索)，将知识库同步到 Milvus (用于 RAG)。

*   **逻辑**:
    1.  **Elasticsearch 同步**:
        *   创建索引 `product_index`，定义 mapping (包含 `dense_vector` 字段)。
        *   调用 Embedding API (OpenAI/DeepSeek) 将 `description` 转为向量。
        *   写入文档: 包含 `name`, `price`, `category`, `tags` (keyword), `embedding`。
    2.  **Milvus 同步**:
        *   创建 Collection `knowledge_base`。
        *   将 `reviews` 和 `specs` 文本切片 (Chunking)。
        *   调用 Embedding API 转为向量。
        *   写入向量及对应的 `product_id` 和原始文本 `text`。

---
**执行顺序**:
1. 运行 `generate_mock_data.py` -> 得到 `data/mock_data.json`
2. 运行 `init_mysql.py` -> 填充 MySQL
3. 运行 `sync_to_vector_db.py` -> 填充 ES 和 Milvus
