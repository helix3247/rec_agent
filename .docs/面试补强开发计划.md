## 二、阶段 9.0：核心代码重构与 Bug 修复 (首要任务)

**面试价值**：消除明显的"AI 生成痕迹"（如长篇重复代码、看起来对但跑不通的逻辑），确保在白板编程或技术深挖环节不露馅。

### 9.0.1 致命逻辑 Bug 修复 (P0)
**文件**：`app/api/endpoints/chat.py`、`app/core/reliability.py`
**任务**：
- [ ] **修复幂等保护完全失效的问题**：`chat.py` 中 `params_hash` 应该只基于 `thread_id` 和 `query`，**不要**包含每次请求都新生成的 `trace_id`。否则每次 hash 都不一样，缓存永远无法命中。
- [ ] **修复熔断器状态机不规范**：`CircuitBreaker` 的 `HALF_OPEN` 状态目前允许多个请求通过，应修改为仅放行**单个探测请求**，成功后再切回 `CLOSED`。

### 9.0.2 异步与并发架构修复 (P0)
**文件**：`app/agents/*.py`、`app/api/endpoints/chat.py`
**任务**：
- [ ] **消除事件循环阻塞**：当前所有 Agent 节点均为同步函数，但在 `chat.py` 中被 `ainvoke` 异步调用。这会导致 FastAPI 并发性能严重受损。需将所有 Agent 节点改为 `async def`，并将内部的 `llm.invoke()` 替换为 `await llm.ainvoke()`。
- [ ] **解决多线程竞态条件**：为 `knowledge.py` 和 `memory.py` 中的全局变量 `_connected`（Milvus连接状态）加上 `asyncio.Lock` 或 `threading.Lock`。

### 9.0.3 DRY 原则重构（消除 AI 复制粘贴痕迹）(P1)
**文件**：`app/core/llm.py`、`app/tools/search.py`、`app/tools/knowledge.py`、`app/agents/*.py`
**任务**：
- [ ] **消除同步/异步代码大面积重复**：`llm.py` 和 `reliability.py` 中的同步/异步重试逻辑几乎逐行重复。重构提取公共逻辑。
- [ ] **客户端复用**：`search.py` 和 `knowledge.py` 中每次调用都创建新的 ES 和 OpenAI 客户端。这在生产中会导致连接数爆增，必须改为单例模式或连接池。
- [ ] **抽取公共 Embedding 函数**：将两个文件中完全重复的 `_get_embedding()` 抽取到 `app/core/embedding.py`。
- [ ] **抽取 Agent 路由模板代码**：提取各 Agent 中高度重复的智能路由和降级逻辑（`classify_complexity -> select_model -> get_llm -> fallback`），考虑使用装饰器或基类重构。

### 9.0.4 测试用例与规范修复 (P2)
**文件**：`tests/test_intent.py`、`tests/conftest.py`、`app/tools/personalization.py`
**任务**：
- [ ] 将 `test_intent.py` 改写为标准的 pytest 格式，并 Mock 掉真实 LLM 调用，避免消耗 Token 和网络依赖。
- [ ] 清理 `conftest.py` 中未使用的 `mock_config` 和 `mock_embedding` fixtures。
- [ ] 修改 `personalization.py`，避免就地（in-place）修改传入的 `products` 列表引起不可预期的副作用。

## 三、阶段 9.1：流式输出（Streaming）

**面试价值**：2026 年 AI 应用不支持 streaming 会被直接质疑技术代差。

### 9.1.1 API 与 Graph 流式适配
**文件**：`app/api/endpoints/chat.py`、`app/graph.py`
**任务**：
- [ ] 新增 `POST /api/chat/stream` 端点，返回 `StreamingResponse`（SSE 格式）。
- [ ] 设计 SSE 事件：`token`、`candidates`、`suggestions`、`done`、`error`。
- [ ] 使用 `app_graph.astream_events` 替代 `ainvoke`，ResponseFormatter 节点改为 `await llm.astream()` 逐 token yield。
- [ ] 确保断连时清理资源，且 MonitorAgent 指标采集在 `finally` 块中正常上报 Langfuse。
