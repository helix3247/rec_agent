# 系统架构优化方案：泛化扩展性设计

## 背景
当前 Rec Agent 采用“特化 Agent”模式（如 [outfit.py](file:///d:/Project/rec_agent/app/agents/outfit.py) 专职穿搭，[shopping.py](file:///d:/Project/rec_agent/app/agents/shopping.py) 专职导购）。随着业务扩展（如新增“美妆推荐”、“家庭影院组建”等），该模式面临以下挑战：
1. **代码冗余**：相似的“组合推荐”逻辑重复开发。
2. **扩展成本高**：每新增一个垂类需开发完整的 Agent、Prompt 和路由规则。
3. **维护复杂度**：分散的 Agent 难以统一优化核心流程。

## 优化目标
构建**配置驱动的泛化架构**，实现：
1. **零代码扩展**：通过配置文件即可接入新垂类（如美妆、健身计划）。
2. **统一调度**：将“任务拆解-检索-组合”逻辑抽象为通用 Agent。
3. **复杂规划**：利用 Planner Agent 实现跨领域的异质任务编排。

---

## 方案一：通用组合专家 (CombinationAgent)

针对“穿搭”、“美妆”、“电脑组装”等**同质化组合任务**，抽象出通用的 `CombinationAgent`。

### 1. 核心设计
不再为每个垂类编写 Python 代码，而是通过**领域配置文件 (Domain Config)** 驱动通用逻辑。

### 2. 配置文件示例 (`config/domains.yaml`)
```yaml
# 领域定义
domains:
  # 穿搭推荐（现有能力迁移）
  outfit:
    name: "穿搭推荐"
    description: "为用户搭配服装、鞋帽和配饰"
    # 任务拆解组件
    components: ["上衣", "裤子", "鞋子", "配饰"]
    # 场景映射规则
    scenario_mapping:
      "通勤": ["商务", "简约", "百搭"]
      "约会": ["时尚", "精致", "优雅"]
    # Prompt 模板路径
    prompt_template: "prompts/templates/combination_tpl.txt"
    # 角色设定
    role_description: "你是一位时尚穿搭顾问，擅长根据场合搭配全身造型..."

  # 美妆推荐（新扩展能力）
  makeup:
    name: "美妆推荐"
    description: "根据肤质和场合推荐完整妆容"
    components: ["妆前乳", "粉底", "遮瑕", "眼影", "口红"]
    scenario_mapping:
      "面试": ["自然", "大地色", "清透"]
      "派对": ["闪亮", "浓郁", "持久"]
    prompt_template: "prompts/templates/combination_tpl.txt"
    role_description: "你是一位专业化妆师，擅长打造适合不同肤质和场合的妆容..."
```

### 3. 工作流程
1. **意图识别**：[intent_parser.py](file:///d:/Project/rec_agent/app/agents/intent_parser.py) 识别出 `combination` 意图及 `domain` 槽位（如 `domain=makeup`）。
2. **加载配置**：`CombinationAgent` 读取 `makeup` 配置。
3. **动态拆解**：遍历 `components`（粉底、眼影...），并行调用 `search_products`。
4. **统一生成**：加载通用 Prompt 模板，填入角色设定和检索结果，调用 LLM 生成方案。

## 方案二：高阶规划师 (PlannerAgent)

针对“去西藏旅游”、“办一场派对”等**异质化复杂任务**，利用 `PlannerAgent` 进行动态编排。

### 1. 定位
Planner 不直接处理商品，而是作为**元 Agent (Meta-Agent)** 调度其他专家 Agent。

### 2. 执行逻辑
当用户提问：“我下周要去三亚拍婚纱照，帮我准备一下。”

**Planner 的执行步骤：**
1. **环境感知**：调用 `WeatherTool` 查询三亚下周天气（得到“晴，30℃”）。
2. **任务分发 (Sub-task Dispatch)**：
   - **任务 A**：调用 `CombinationAgent(domain='outfit')`
     - 指令：“推荐适合30度海滩的婚纱照穿搭，风格轻婚纱”
   - **任务 B**：调用 `CombinationAgent(domain='makeup')`
     - 指令：“推荐适合海边、防水防汗的持久妆容”
   - **任务 C**：调用 `RAGAgent`
     - 指令：“海边拍照有哪些注意事项？如何防晒？”
3. **结果汇总**：整合上述 Agent 的输出，生成最终的各种建议合集。

## 数据层配套改造

### 1. 商品数据结构泛化
Elasticsearch 中的商品索引需支持多垂类：
- **Category 字段**：标准化为各垂类的组件名（如“粉底”、“显卡”）。
- **Tags 字段**：建立垂直领域的标签体系（美妆：油皮/干皮；电脑：兼容性/功耗）。
- **Specs 字段**：存储特有属性（色号、SPF值、芯片架构）。

### 2. 知识库扩容
Milvus 需按 `domain` 隔离或打标，存储各领域的专家知识（评测、教程、避坑指南）。
